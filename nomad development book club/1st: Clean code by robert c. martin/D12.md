# 2022.02.01. Tue `(Objects and Data Structures)`

## TL; DR

```markdown
- 자료는 추상적인 개념으로 표현하는 것이 좋다
- 새로운 자료 타입을 추가하기 위한 유연성은 객체가 새로운 동작을 추가하기 위한 유연성은 자료 구조와 절차적인 코드가 더 좋다
- 우수한 개발자는 편견없이 직면한 문제에 최적의 해결책을 찾아 해결한다
```

